# Key Concepts 
* **Syntax Difference**
  * **HTML (Vanilla JS):**
    ```html
    <button onclick="doSomething()">Click me</button>
    ```
  * **React (JSX):**
    ```jsx
    <button onClick={doSomething}>Click me</button>
    ```
* âœ… Note: React uses camelCase for event names (e.g., onClick, not onclick)
* âœ… The event handler is passed as a function, not a string

# Examples
* **onClick**
  ```jsx
  function App() {
  const handleClick = () => {
    alert('Button clicked!');
  };
  
  return <button onClick={handleClick}>Click Me</button>;
  }
  ```
* **onChange (Input Field)**
  ```jsx
  function App() {
  const [name, setName] = React.useState('');

  return (
    <input
      type="text"
      value={name}
      onChange={(e) => setName(e.target.value)}
    />
  );
  }
  ```
* **onSubmit (Form)**
  ```
  function App() {
  const handleSubmit = (e) => {
    e.preventDefault(); // Prevent page reload
    alert('Form submitted!');
  };

  return (
    <form onSubmit={handleSubmit}>
      <button type="submit">Submit</button>
    </form>
  );
  }
  ```


# ðŸ“˜ React Events Cheatsheet

React wraps browser events in a cross-browser wrapper called **SyntheticEvent**. You handle events in JSX using camelCase and pass a function, not a string.

## ðŸ”¹ Common Event Types

### ðŸ”¸ `onClick`
```jsx
<button onClick={() => alert('Clicked!')}>Click</button>
```

### ðŸ”¸ `onChange` (Input)
```jsx
<input type="text" onChange={(e) => console.log(e.target.value)} />
```

### ðŸ”¸ `onSubmit` (Form)
```jsx
<form onSubmit={(e) => {
  e.preventDefault();
  console.log('Form submitted');
}}>
  <button type="submit">Submit</button>
</form>
```

### ðŸ”¸ `onKeyDown`
```jsx
<input onKeyDown={(e) => console.log('Key:', e.key)} />
```

### ðŸ”¸ `onFocus` and `onBlur`
```jsx
<input onFocus={() => console.log('Focused')} onBlur={() => console.log('Blurred')} />
```

### ðŸ”¸ `onMouseEnter` and `onMouseLeave`
```jsx
<div onMouseEnter={() => console.log('Mouse entered')} onMouseLeave={() => console.log('Mouse left')}>
  Hover me
</div>
```

---

## âœ… Notes

- Event names use **camelCase**: `onClick`, `onSubmit`, etc.
- Event handlers are **functions**, not strings.
- React events are **synthetic** but you can access native events via `e.nativeEvent`.
